需求场景	推荐范式	原因
简单信息检索	ReAct	效率与灵活性平衡
复杂工作流	Plan-and-Execute	结构化执行保障
跨领域问题	MRKL	领域专业化路由
高质量内容生成	Reflexion	迭代优化提升质量
精确API调用	Toolformer	工具调用精准控制

情感交互场景	Emotion-Aware	情感智能增强用户体验

Agent范式对比
1. ReAct（推理-行动）
核心：循环执行"思考→行动→观察"

最佳场景：问题求解/信息检索

特点：平衡灵活性与效率

2. Plan-and-Execute（规划执行）
核心：先制定完整计划再逐步执行

最佳场景：复杂多步骤任务（如数据处理流水线）

特点：结构化程度高，适合确定性强任务

3. MRKL（模块化路由）
核心：路由到专业领域模块

最佳场景：跨领域问题求解

特点：发挥领域专家优势

4. Reflexion（自我反思）
核心：生成→批评→改进迭代循环

最佳场景：内容创作/代码生成

特点：输出质量高，迭代成本高

5. Toolformer（工具学习）
核心：LLM学习精确插入工具调用

最佳场景：API密集型任务

特点：工具调用精准可控

6. Democratic（民主投票）
核心：多专家并行+共识决策

最佳场景：高可靠性决策系统

特点：可靠性高，资源消耗大

7. Emotion-Aware（情感感知）
核心：情感状态驱动响应

最佳场景：客服/心理咨询

特点：增强交互自然度
